Importing JUMP Library on your XCode project

JUMP is divided on several modules, so you doesn't have to import all modules at the same time, only those that you needed it. Also some modules already depends of each other, so sometimes when you import one module you're importing all the dependecies.

Check JUMP Module Dependecies first, so you'll known what you're importing and don't import the same module twice.

Let's see how to import the JUMP Logger module:

  • Open your XCode project, is a good practice to create an Dependecies folder inside the project.
  • Now go to the JUMP/JUMPLogger folder, locate the JUMPLogger.xcodeproj file and drag this file inside your XCode project.
  • Select the project you just imported. On XCode Main Menu select View > Zoom Editor Out. Now check the Target Icon Checkbox next to the libJUMPDatabase.a. This will automatically associate the library to your Target under the Link Binary With Libraries section.

The next step is to inform the XCode where to find the library Headers. If you remeber, when we configure the library one Headers folder was created. There is where the XCode will look for all JUMP Headers, let's do this:

  • Open your XCode Target Info and go to the Build tab.
  • Under the Build tab look for the Header Search Pahts section. Double-click and create a new entry. Now paste the JUMP/Headers full path there.
  • Also in the Build tab look for the Other Linker Flags section. Set the -all_load flag if isn't already setted.
  • Now under the General tab we need to create a new Direct Dependency for our library. Click on the + button and select the JUMPLogger that have the Static Library Icon. Add to the target.



That's it, try to compile your project and everything should work fine. You can repeat this steps for every JUMP Module that you need to use on your project.
