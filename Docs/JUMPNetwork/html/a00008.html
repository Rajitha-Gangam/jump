JSON-RPC Event Messages

JPJSONRPCMessage is an subclass of JPDefaultHTTPMessage and implements the JPTransporterJSONRPCMessage protocol, it is designed to transport JSON-RPC Event Messages on top of the HTTP protocol.

You can use this class directly, create your own subclass of this event or create your own implementation of the JPTransporterJSONRPCMessage protocol.

An simply code to illustrate how to send an JPTransporterJSONRPCMessage downstream: 

 JPJSONRPCMessage *eventMessage = [JPJSONRPCMessage initWithMethod:@"POST"]
 eventMessage.transportURL = [NSURL URLWithString:@"http://seqoy.org/httpgateway"];
 [eventMessage setMethod:@"someCall" 
           andParameters:[NSArray arrayWithObjects:@"parameter1", @"parameter2", nil]
                   andId:[NSNumber numberWithInt:2]];
 
 [pipeline sendDownstream:[JPPipelineMessageEvent initWithMessage:eventMessage]];

Of course this example require a big boilerplate for a simple operation. See Using Factories for more information to how configure and automate boilerplates using Factory Patterns.

JSON-RPC Encoder and Decoder

JSON-RPC Event Messages are processed by they respectives JSON-RPC Handlers that decode and encode the Event Message upstream or downstream respectively.

Encoder

JPJSONRPCEncoderHandler intercepts downstream JPJSONRPCMessage and encode his RPC properties, fisrt on a JSON String and later on NSData to be sent by a I|O Transporter. Other types of messages are ignored and sented downstream to the next handler on the The Pipeline.

Here an example that how you assign the JPJSONRPCEncoderHandler to the pipeline: 

 [pipeline addLast:@"JSONRPCEncoder" withHandler:[JPJSONRPCEncoderHandler initWithJSONEncoderClass:[JSONEncoder class]]];

The JPJSONRPCEncoderHandler doesn't have an embedded JSON Encoder. You have to inform one JSON Processer Class that conform with the JPDataProcessserJSON protocol. See the JUMP Data Module to find an default implementation of this protocol that you can use or you can implement your own.

Decoder

JPJSONRPCDecoderHandler intercepts any upstream Event Message and try to decode as a JSON Object first, and then to interpret the JSON-RPC properties.

If JPJSONRPCDecoderHandler can't decode to JSON Object or can't found the RPC properties will send an JPDefaultPipelineExceptionEvent upstream as a warning error, you can catch this exception and process it as an decode error. Refer to JPJSONDecoderErrors to known the proper error code.

You also can ignore this exception and let the next handler (if exist) continue the processing, because the unmodified Event Message are sent upstream to the next handler even when an error ocurrs. This is useful if you have an pipeline that decode differents types of Event Message (e.g. XML and JSON) at the same time.

Here an example that how you assign the JPJSONRPCDecoderHandler to the pipeline: 

 [pipeline addLast:@"JSONRPCDecoder" withHandler:[JPJSONRPCDecoderHandler initWithJSONDecoderClass:[JSONEncoder class]]];

The JPJSONRPCDecoderHandler doesn't have an embedded JSON Decoder. You have to inform one JSON Processer Class that conform with the JPDataProcessserJSON protocol. See the JUMP Data Module to find an default implementation of this protocol that you can use or you can implement your own.
